## About

MERN-auth-roles a full-stack MERN (MongoDB Express React Node) boilerplate starter application with React, Recoil, authentication, roles, JWT, protected api
![MERN-auth-roles](https://i.imgur.com/7J6c8Rf.png)

We have two parts in the application

## MERN-AUTH-ROLES-Backend (sever)

### Futures

‚úì User with roles (Admin, Manager, user)

‚úì NodeJS server

‚úì Token and refresh token

‚úì async/await syntax

‚úì Server side validation

‚úì .env file configuration

‚úì Profile image upload with Multer (delete old image and replace it with the new image),

‚úì Cross-origin resource sharing (CORS)

‚úì Limit repeated requests such as password reset.

‚úì Cookies

‚úì Password Hashing

‚úì Real-time notifications using Socket.io

run this command

```
  cd backend
  npm install
  npm run dev
```

## MERN-AUTH-ROLES-Frontend (client)

### Futures

‚úì React DevTools desabled in production

‚úì Axios with Interceptors to manage fetchs

‚úì Token persist only in memory and cookies

‚úì Refresh Token (in memory) and access token (in cookies)

‚úì Hookform: to manage form inputs

‚úì YUP: to validate inputs

‚úì react-multi-select-component

‚úì Recoil and Recoil-persist: to manage states

‚úì react-icons and @uiball/loaders to give a nice look to the UI

‚úì Protected routes with Higher order components

‚úì Layout component, it will be very easy to navigation and footer

‚úì Profile image upload

‚úì Real-time notifications using Socket.io client

### Run the code

you can run the code by executing this command

```
cd frontend
npm install
npm start
```
